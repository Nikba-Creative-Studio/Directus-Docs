# App Admin Guide

This guide is written for administrators looking to configure Directus. It is meant to cover setup after installation. If you still need to install Directus, you can follow our dedicated section on Installing Directus.

Administrators Only

Administrators are the only role with access to the Directus project settings, which can be accessed by clicking the "gear" icon in the module sidebar.

# Global Settings

The general settings for your project.

# Project

  • Project Name ‚Äì Identifies this project on the login page and project switcher
  • Project URL ‚Äì Clicking the top-left logo square takes you here
  • Project Logo ‚Äì A 40x40 pixel brand logo. Ideally an SVG or PNG with alpha layer
  • Project Color ‚Äì A hex value for the top-left logo background and login background color
  • Project Foreground ‚Äì A centered image shown on the login page when the project is selected
  • Project Background ‚Äì A full-screen image shown on the login page when the project is selected
  • Project Public Note ‚Äì A short markdown note shown on the login page when the project is selected
  • Default Locale ‚Äì The default locale for users. Users can individually override this as needed
  • Telemetry ‚Äì Toggles the anonymous Directus telemetry information from being sent

# Data

  • Default Limit ‚Äì Default item count returned for App and API responses
  • Sort NULL Last ‚Äì NULL values are sorted last

# Security

  • Auto Sign Out ‚Äì The number of minutes before an idle user is logged out
  • Login Attempts Allowed ‚Äì The number of failed login attempts before a user's status is set to suspended. Once suspended, the user can not log in and an administrator must unlock the account.
  • Password Policy ‚Äì Saves a RegEx new user passwords must match
    • None: No rules are enforced
    • Weak: Minimum length 8 characters
    • Strong: Minimum length 8 characters with at least 1 lowercase letter, 1 capital letter, 1 digit, 1 special character

Suspended Administrators

If your project only has one administrator and they become suspended after too many failed login attempts, the only way to re-activate them is directly through the database.

  • Learn more about Authentication

# Files & Thumbnails

  • File Naming ‚Äì The file-system naming convention used for uploads
    • UUID (Obfuscated): A universally unique identifier that helps keep files private
    • File Name (Readable): A human readable name based on the original upload's title
  • File MimeType Whitelist ‚Äî The mimetypes allowed to be uploaded. If empty, all types are allowed.
  • Asset Whitelist ‚Äî Defines the details of image assets that can be dynamically generated. Learn more.
  • YouTube API Key ‚Äî When added, this allows Directus to fetch more metadata for YouTube embeds.
  • Learn more about Files & Thumbnails
